The Targets Namespace




This namespace contains everything related to the logging targets. In the namespace `targets`, you will find:
  • namespace helpers
    - This nested namespace includes the following helpers used for all in-house targets:
    • BaseTargetHelper - This class contains the multi-threading support functions and flush policy options that Serenity's targets use and is composed from (NOT inherits from) the following classes:
      • BackgroundThread
      • Flush_Policy
    • FileHelper
      - This class aids Serenity's targets in file related operations. This includes things such as:
      • Opening and closing the file
      • Writing to the buffer
      • Setting the file buffer size
      • Flushing the buffer to the file
      • Controlling the background thread worker in charge of flushing the buffer
      • Renaming the log file
      This class is composed from (NOT inherits from) the following classes:
      • FileCache
      • BaseTargetHelper
      • BackgroundThread
  • TargetBase
    - Base class all targets inherit from. This class is composed from (NOT inherits from) the following classes:
    • BaseTargetHelper
    • Message_Formatter
    • Message_Info
  • ConsoleTarget
    - Class that handles terminal logging. This class inherits from the following:
    • TargetBase
    and is composed from the following:
    • console_interface
  • FileTarget
    - Class that handles logging to a singular file. This class inherits from the following:
    • TargetBase
    • FileHelper
  • RotatingTarget
    - Class that handles logging to multiple files and under conditions based on user settings. This class inherits from the following:
    • TargetBase
    • RotateSettings
    and is composed of the following:
    • RotatingDaylightCache
    • IntervalMode
  • RotatingDaylightCache
    - Class that handles updating the background thread that runs when a time setting is used with RotatingTarget
